# Focused Action Plan for the Experimental Elemento Website

Use this document as the execution backlog for vibecoding agents.

The current experimental site already has the correct Metacloud skeleton:

- Electros is present.
- C4 is present.
- Atomosphere is present.
- Atomosquare is present.
- AtomOS is present.
- The positioning already uses “The Infrastructure Layer After Cloud.”

The problem is execution.

The homepage explains too much too early, the commercial hooks are weak, the wow factor is missing, and some pricing/licensing claims need to be updated to match the new product direction.

## 0. Core Implementation Principle

### Main site

Sell the Metacloud category and route visitors to the right conversion path.

### electros.cloud

Convert buyers who want a control plane.

### atomos.cloud

Convert technical users who want a free sovereign hypervisor and support.

### BookStack / docs

Explain technical depth.

### GitHub

Build open-source trust around AtomOS and C4.
