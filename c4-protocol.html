Cloud Compute Common Connector Protocol

C4 Protocol

The Common Language of the Metacloud. C4 is the interoperability framework that allows heterogeneous infrastructure platforms to participate in one federation model.

Read the C4 Technical Brief View Architecture

Why C4 Exists

C4 does not replace provider APIs.

It provides a common language that translates federation operations into provider-specific operations, preserving existing technology choices while enabling a unified operational framework.

Declare an outcome through portable C4 contracts. The Metacloud identifies and activates the most suitable implementation.

Contracts, not clouds.

Core Functions

Resource abstraction

Normalise heterogeneous provider capabilities into a common model.

Service discovery

Make infrastructure and services discoverable across the federation.

Endpoint registration

Register providers, hypervisors and services into the Metacloud.

Capability exposure

Expose provider-specific features through a unified interface.

Common API model

One operational language for Electros across all connected environments.

Workload operations

Provision, move and govern workloads through federation contracts.

Design Principle

C4 acts as a translation layer between Electros and provider-specific APIs.

Each provider exposes its capabilities through C4, allowing Electros to issue common federation operations without requiring knowledge of the underlying provider implementation.
